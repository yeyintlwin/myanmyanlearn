# 1-5 　問題解決の方法 ①（アルゴリズム）

問題解決をするのに、また、コンピューターに間違いのないプログラムを実行させるのに必要なアルゴリズムについて学習します。

---

> 問題解決

## (1) コンピュータの特性

コンピュータは、プログラムを忠実に実行するだけで、 以外のことはできません。

## (2) アルゴリズムの重要性

コンピュータに処理をさせる場合は、正確なプログラムを用いないと正確な処理はできません。

そこで、プログラムのもととなる問題解決のための手順や考え方をあらわす**アルゴリズム**が重要になります。

アルゴリズムとは、問題を解決するために組み立てた処理手順です。

---

> 問題解決の手順

## (1)アルゴリズム

朝起きて学校へ行くまでのアルゴリズムの例は、次のようになります。

|               |
| :------------ |
| 1．顔を洗う   |
| 2．朝食をとる |
| 3．着替える   |
| 4．家を出る。 |
| 5．駅へ行く   |
| 6．電車に乗る |
| 7．学校に着く |

結果は同じであっても、行動(処理手順)は人によってさまざまです。コンピュータは、処理手順の良し悪しが処理効率を大きく左右するため、最良の処理手順、すなわちアルゴリズムを考え出さなければなりません。

## (2)アルゴリズムの表記方法

アルゴリズムを表現する方法がばらばらでは、書いた人以外は理解できません。そこで、**フローチャート** (流れ図) や**PAD**(問題解析図)などが、アルゴリズムを表現するために使われます。

代表的な流れ図記号を使って説明してみます。
