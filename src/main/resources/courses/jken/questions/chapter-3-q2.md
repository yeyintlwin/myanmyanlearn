- <p>会社内の複数のパソコン間で通信するために LAN を構築する。複数あるパソコンのうち、どのパソコン宛に通信するのかを区別するために、各パソコンに異なるアドレスが必要になる。通常、パソコンを LAN に接続するには、パソコン内にある <span style="border:1px solid #000; padding: 0 1em 0 1em ">1</span> と呼ばれる拡張カードに LAN ケーブルを接続する。この <span style="border:1px solid #000; padding: 0 1em 0 1em ">1</span> には、<span style="border:1px solid #000; padding: 0 1em 0 1em ">2</span> と呼ばれる固有のアドレスが付けられている。しかし、<span style="border:1px solid #000; padding: 0 1em 0 1em ">2</span> は <span style="border:1px solid #000; padding: 0 1em 0 1em ">1</span> 製造時に、48 ビットの 2 進数で ROM に記録されるため、各パソコンのアドレスが分かりづらい。そこで LAN 構築後に、このパソコンは 1 番、このパソコンは 2 番というように、人間が管理して各パソコンに後から付与するアドレスが <span style="border:1px solid #000; padding: 0 1em 0 1em ">3</span> である。もちろん 1 番 2 番という単純なものではないが、ネットワークを構築する側の意図をもって割り当てられる。図 1 のような LAN で、データの送信が行われると通信路上にデータは流れるが、各パソコンはアドレスにより自分宛か否かを判断して、正しい受け手だけがデータを取り込むことになる。</p>

![local_image](../images/c3q2_fig1.jpg)

- <p>また、複数のパソコンをまとめて LAN に接続するときは、図 2 のような装置が使用される。装置 1 は <span style="border:1px solid #000; padding: 0 1em 0 1em ">4</span> と呼ばれ、1 台の PC から送信されたデータは、他のすべての PC や LAN に送られる。これに対して装置 2 は <span style="border:1px solid #000; padding: 0 1em 0 1em ">5</span> と呼ばれ、<span style="border:1px solid #000; padding: 0 1em 0 1em ">2</span> を判断して、特定の PC だけにデータを送ることができる。</p>

![local_image](../images/c3q2_fig2.jpg)

- <p>さらに、図 3 の装置 3 のように部署ごとに LAN1、LAN2 と割り当てて、部署内の PC 同士の通信が他の部署に流れないようにする機能を持つ装置は <span style="border:1px solid #000; padding: 0 1em 0 1em ">6</span> と呼ばれ、<span style="border:1px solid #000; padding: 0 1em 0 1em ">3</span> を利用した経路選択の機能を持っている。<p/>

![local_image](../images/c3q2_fig3.jpg)

- <p>また、図 4 のように、ケーブルを使わず、無線で LAN に接続することもできる。このときの装置 4 は <span style="border:1px solid #000; padding: 0 1em 0 1em ">7</span> と呼ばれている。</p>

![local_image](../images/c3q2_fig4.jpg)
